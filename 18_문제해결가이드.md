# 18_문제해결가이드: 일반적인 문제와 해결 방법

워크플로우 실행 중 발생할 수 있는 일반적인 에러들과 디버깅 팁을 정리합니다.

---

## 1. 자주 발생하는 에러 (Common Errors)

### 401 Unauthorized / 403 Forbidden
- **원인**: API 키가 만료되었거나 권한이 부족함.
- **해결**: Credentials 설정을 다시 확인하고, 해당 서비스의 API 대시보드에서 키 상태와 Scope를 확인하세요.

### Timeouts (Execution Timeout)
- **원인**: 노드 처리가 너무 오래 걸리거나 응답이 없는 서버를 기다림.
- **해결**: 노드 설정의 `Settings > Timeout`을 늘리거나, `Split In Batches`를 사용하여 작업 단위를 나누세요.

### Memory Limit Reached
- **원인**: 너무 큰 JSON 데이터나 바이너리 파일을 메모리에 로드함.
- **해결**: `N8N_MAX_PAYLOAD_CONTENT_LENGTH` 환경 변수를 조정하거나 시스템 메모리를 증설하세요. 데이터 최소화 전략을 사용하세요.

---

## 2. 디버깅 기술 (Debugging Tips)

### 실행 히스토리 (Execution History)
- 실패한 실행 건을 클릭하여 각 노드의 **Input**과 **Output**을 대조해 보세요. 데이터가 어디서 변질되었는지 찾는 가장 빠른 방법입니다.

### Debug 노드 만들기
- `No-Op` (도움말: Edit Data) 노드를 중간중간 배치하여 중간 결과물을 확인하는 용도로 사용하세요.

### 로그 파일 확인
- n8n 서버 로그(Docker log 등)를 통해 워크플로우 외부의 시스템 에러(DB 연결 실패, 네트워크 단절 등)를 확인하세요.
