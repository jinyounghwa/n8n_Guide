## n8n이란?

**n8n**(엔테인)은 다음의 약자입니다: **n** + **8** + **n** (n이 8개 있다는 뜻)

n8n은 **노드 기반 자동화 플랫폼**입니다.

### 간단한 정의

> 여러 서비스와 API를 연결하여 자동화된 워크플로우를 만드는 도구

### 현실적인 예시

```
Gmail 수신 → 데이터 추출 → Claude AI 분석 → Notion 저장 → Slack 알림
```

이런 복잡한 작업을 노드를 연결하는 것만으로 자동화할 수 있습니다.

---

## n8n의 특징

### 1. 오픈소스
- 무료로 사용 가능
- 로컬에서 설치 가능
- 소스코드 공개

### 2. 낮은 진입장벽
- UI 기반 (코딩 최소화)
- 시각적으로 워크플로우 설계
- 초보자도 사용 가능

### 3. 강력한 확장성
- 600+ 통합 가능한 서비스
- Custom Code 지원 (JavaScript)
- Webhook으로 외부 연결

### 4. 안전성
- 로컬 실행 가능
- 데이터는 자신의 서버에만 저장
- 오픈소스로 안심

## n8n의 핵심 개념

### 1. 워크플로우 (Workflow)
자동화되는 작업 흐름 전체

```
┌─────────────────────────────────────┐
│      나의 첫 워크플로우              │
├─────────────────────────────────────┤
│ [Webhook] → [HTTP] → [Set] → [Slack]│
└─────────────────────────────────────┘
```

### 2. 노드 (Node)
각각의 작업 단계

```
노드 타입:
- 트리거: 워크플로우를 시작하는 노드 (Webhook, Schedule 등)
- 액션: 실제 작업을 수행하는 노드 (HTTP, Gmail, Slack 등)
- 로직: 데이터를 처리하는 노드 (Set, If, Loop 등)
```

### 3. 연결 (Connection)
노드 간의 데이터 흐름

```
노드1 ──(데이터)──> 노드2 ──(데이터)──> 노드3
```

### 4. 데이터 (Data)
노드 간에 전달되는 정보

```json
{
  "name": "John",
  "email": "john@example.com",
  "age": 25
}
```

## n8n의 동작 원리

### 기본 흐름

```
┌───────────────────────────────────────────┐
│1. 트리거 발생                              │
│   (Webhook, Schedule, Database 등)        │
└─────────────┬───────────────────────────┘
              │
              ▼
┌───────────────────────────────────────────┐
│2. 워크플로우 시작                         │
│   (메모리에 초기 데이터 로드)             │
└─────────────┬───────────────────────────┘
              │
              ▼
┌───────────────────────────────────────────┐
│3. 각 노드 순차 실행                       │
│   ├─ 입력 데이터 받음                     │
│   ├─ 처리 수행                           │
│   ├─ 출력 데이터 생성                     │
│   └─ 다음 노드에 전달                     │
└─────────────┬───────────────────────────┘
              │
              ▼
┌───────────────────────────────────────────┐
│4. 완료 또는 에러                          │
│   ├─ 성공: 로그 저장                      │
│   └─ 실패: 에러 처리 (Retry 등)          │
└───────────────────────────────────────────┘
```

### 데이터 흐름 예시

```
[Webhook 트리거]
    ↓ (데이터: {email: "user@example.com"})
[HTTP 요청]
    ↓ (데이터: {email, user_data: {...}})
[데이터 변환]
    ↓ (데이터: {email, processed: {...}})
[Slack 메시지]
    ↓ (완료)
```

## n8n의 장점과 단점

### ✅ 장점

1. **빠른 개발**: 코드 작성 없이 빠르게 자동화 구현
2. **비용 효율**: 오픈소스로 무료 사용 가능
3. **자유도**: 로컬 실행으로 완전한 제어
4. **확장성**: 600+ 통합 + Custom Code 지원
5. **학습하기 쉬움**: UI 기반으로 진입장벽 낮음
6. **커뮤니티**: 활발한 오픈소스 커뮤니티

### ❌ 단점

1. **설치 필요**: 로컬에 설치/관리 필요
2. **성능 제약**: 대규모 워크플로우는 성능 최적화 필요
3. **에러 처리**: 복잡한 에러 처리는 수동 관리 필요
4. **모니터링**: 프로덕션 모니터링 도구 필요
5. **학습 곡선**: 고급 기능은 JavaScript 이해 필요

## 왜 n8n을 배우는가?

### 개발자 관점에서

1. **시간 절약**: 반복적인 작업 자동화
2. **통합 경험**: 다양한 API/서비스 통합 이해
3. **프로토타이핑**: 빠른 프로토타입 개발
4. **부가 수익**: 클라이언트를 위한 자동화 솔루션 개발
5. **AI 활용**: Claude API 등 AI 서비스 통합 경험

### 기업 관점에서

1. **효율성 증대**: 수동 작업 자동화
2. **비용 절감**: 개발 비용 감소
3. **에러 감소**: 자동화로 인한 휴먼 에러 제거
4. **확장성**: 새로운 통합 쉽게 추가
5. **생산성**: 팀이 더 중요한 업무에 집중

## 핵심 용어 정리

| 용어 | 설명 |
|------|------|
| **Workflow** | 자동화 작업 흐름 전체 |
| **Node** | 각각의 작업 단계 |
| **Trigger** | 워크플로우를 시작하는 노드 |
| **Action** | 실제 작업을 수행하는 노드 |
| **Credentials** | API 키, 인증 정보 |
| **Expression** | 동적 값을 생성하는 식 |
| **Mapping** | 한 노드의 출력을 다음 노드의 입력으로 연결 |
| **Execution** | 워크플로우 실행 |
| **Execution Mode** | 실행 방식 (Regular, Webhook, Database 등) |
