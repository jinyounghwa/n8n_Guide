# 13_프로젝트2: 고객 응답 자동화 시스템

이 프로젝트에서는 수신된 이메일을 분석하고, Claude API를 통해 적절한 답변 초안을 작성하여 저장하는 워크플로우를 구축합니다.

---

## 1. 프로젝트 개요
- **목적**: 반복되는 고객 문의에 대해 AI가 답변 초안을 생성하여 업무 효율성 증대
- **사용 노드**: 
  - `Gmail` (또는 Email Read nodes): 이메일 수신 트리거
  - `Set`: 필요한 데이터 추출 (발신자, 본문)
  - `Claude API`: 문의 내용 분석 및 답변 생성
  - `Gmail` (또는 Slack): 초안 전송 또는 저장

---

## 2. 워크플로우 구성 단계

### 1단계: 이메일 트리거 설정
- n8n의 `Gmail` 노드를 사용하여 특정 라벨(예: "Inquiry")이 붙은 새 이메일을 감지합니다.
- (테스트 환경에서는 `Imap` 노드나 `Webhook` 노드를 대신 사용할 수 있습니다.)

### 2단계: 데이터 정제
- 메일 본문에서 불필요한 HTML 태그를 제거하고 텍스트만 추출합니다.
- `Set` 노드를 사용하여 `subject`, `sender`, `body`를 정리합니다.

### 3단계: Claude API 프롬프트 작성
- **프롬프트 예시**:
  ```text
  다음은 고객의 문의 이메일 내용입니다. 
  친절하고 전문적인 어조로 답변 초안을 작성해 주세요. 
  문의 내용: {{ $json.body }}
  ```

### 4단계: 답변 초안 전송/저장
- 생성된 답변을 Gmail의 `Create Draft` 노드를 사용하여 "임시 보관함"에 저장하도록 설정합니다.
- 또는 담당자에게 Slack으로 "AI 제안 답변"을 전달합니다.

---

## 3. 학습 포인트
- 이메일 데이터의 구조 이해
- 도메인 특화 프롬프트 작성법 (CS 대응용 어조)
- 자동 발송이 아닌 "초안 저장"을 통한 인간의 최종 확인 프로세스(Human-in-the-loop) 설계
