# 8장: 에러 처리

> 워크플로우에서 발생하는 에러를 처리합니다.
> 예상 학습 시간: 1시간

## Try-Catch 노드

### 기본 구조

```
[위험한 작업]
  ↓
[Try-Catch]
  ├─ Success: 정상 처리
  └─ Error: 에러 처리
```

### 에러 정보 접근

```
{{ $error.message }}     // 에러 메시지
{{ $error.code }}        // 에러 코드
{{ $error.status }}      // HTTP 상태 코드
{{ $error.response }}    // 응답 전체
```

## 일반적인 에러

### HTTP 에러

```
400 Bad Request: 요청 형식 오류
401 Unauthorized: 인증 실패
403 Forbidden: 권한 없음
404 Not Found: 리소스 없음
429 Too Many Requests: 요청 과다
500 Server Error: 서버 에러
```

## 에러 처리 전략

### 1. 재시도

```
Try:
  API 호출
Catch:
  IF 429 또는 500
    → 대기 후 재시도
  ELSE
    → 로그 기록
```

### 2. 폴백

```
Try:
  기본 방법
Catch:
  → 대체 방법 시도
  Catch:
    → 기본값 사용
```

### 3. 알림

```
Try:
  작업
Catch:
  → Slack 알림
  → 이메일 발송
  → 데이터 로깅
```
